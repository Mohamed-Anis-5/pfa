================================================================================
  FOURTH SPRINT REPORT — Municipal Complaints Platform
  Date: April 20, 2026
================================================================================

This sprint delivered the complete frontend application and extended the backend
with complaint listing endpoints, a feedback sub-system, a dashboard service,
a user management endpoint, and several supporting DTOs, entities, and
repositories.

================================================================================
  FRONTEND — NEW FILES
================================================================================

The entire React/Vite frontend is new in this sprint.

------------------------------------------------------------------------
src/main.jsx
  Entry point. Mounts the <App /> component into the DOM root.

src/App.jsx
  Root component.
  - RoleRedirect()          Reads the authenticated user's role and redirects
                            to /citizen, /admin, or /agent accordingly.
  - App()                   Wraps the application in <AuthProvider> and
                            <BrowserRouter>; declares all client-side routes
                            with role-based <ProtectedRoute> guards.

  Routes defined:
    /login               → Login page (public)
    /register            → Register page (public)
    /                    → RoleRedirect (authenticated)
    /citizen             → CitizenDashboard (ROLE_CITIZEN)
    /citizen/submit      → SubmitComplaint (ROLE_CITIZEN)
    /citizen/complaints  → MyComplaints (ROLE_CITIZEN)
    /admin               → AdminDashboard (ROLE_ADMIN)
    /admin/analytics     → Analytics (ROLE_ADMIN)
    /agent               → AgentDashboard (ROLE_AGENT)
    /unauthorized        → 403 error page

------------------------------------------------------------------------
src/api/axios.js
  Configured Axios instance for all API calls.
  - baseURL set to http://localhost:8080/api
  - Request interceptor: automatically attaches the JWT Bearer token from
    localStorage to every outgoing request header.
  - Response interceptor: on HTTP 401, clears localStorage and redirects
    the browser to /login (auto-logout).

------------------------------------------------------------------------
src/context/authContext.js
  Creates and exports the React context object (AuthContext) used to share
  authentication state across the component tree.

src/context/AuthContext.jsx
  Exports the AuthProvider component.
  - AuthProvider({ children })
      Initialises user state lazily from localStorage at mount time (reads
      token, role, email; decodes JWT with jwt-decode). No side-effect hook
      is used, satisfying the React fast-refresh lint rule.
    - login(token, email, role)   Persists credentials to localStorage,
                                  decodes the JWT, and sets user state.
    - logout()                    Clears localStorage and sets user to null.

src/context/useAuth.jsx
  Custom hook.
  - useAuth()   Returns the current AuthContext value (user, login, logout).
    Separated into its own file so AuthContext.jsx exports only a component
    (required by the react-refresh/only-export-components lint rule).

------------------------------------------------------------------------
src/routes/ProtectedRoute.jsx
  - ProtectedRoute({ children, allowedRoles })
      Guards a route: redirects to /login if unauthenticated; redirects to
      /unauthorized if the user's role is not in allowedRoles.

------------------------------------------------------------------------
src/pages/auth/Login.jsx
  - Login()
      Controlled form with email + password fields.
      - handleSubmit(e)   POST /api/auth/login, calls login() from useAuth,
                          then navigates to the role-specific dashboard.

src/pages/auth/Register.jsx
  - Register()
      Controlled form with firstName, lastName, email, password,
      phoneNumber, identifiantUnique, and role selector.
      Includes the full list of 24 Tunisian governorates as a constant.
      - field(label, key, type)   Helper that renders a labelled input.
      - handleSubmit(e)           POST /api/auth/register, redirects to /login.

------------------------------------------------------------------------
src/pages/citizen/CitizenDashbord.jsx
  - CitizenDashboard()
      Welcome page for citizens. Shows two navigation cards: Submit Complaint
      and My Complaints.

src/pages/citizen/SubmitComplaint.jsx
  - SubmitComplaint()
      Multi-field complaint creation form.
      - captureLocation()    Uses the browser Geolocation API to capture the
                             device's latitude/longitude and updates form state.
      - handleSubmit(e)      POST /api/complaints (JSON body), then if a file
                             was selected POST /api/complaints/{id}/attachments
                             as multipart/form-data. Navigates to /citizen/complaints
                             on success.

src/pages/citizen/MyComplaints.jsx
  - MyComplaints()
      Lists all complaints belonging to the logged-in citizen (GET /api/complaints/my).
      Shows title, status badge, category, due date, and assigned agent.
      For RESOLVED complaints shows a "Rate this service" button that reveals
      the StarRating component; for CLOSED complaints shows a confirmation label.
      - handleFeedback(id, payload)   POST /api/complaints/{id}/feedback, then
                                      refreshes the complaint list.

------------------------------------------------------------------------
src/pages/admin/AdminDashboard.jsx
  - AdminDashboard()
      Shows all complaints (GET /api/complaints) and all agents (GET /api/users/agents).
      - assign(complaintId, agentId)   PUT /api/complaints/{id}/assign; refreshes
                                       the list on success.
      Includes filter controls (by status and category), an interactive
      ComplaintsMap, and a sortable table with an inline agent assignment
      dropdown for PENDING complaints.

src/pages/admin/Analytics.jsx
  - Analytics()
      Fetches aggregated statistics (GET /api/dashboard/stats) and renders:
      * Average resolution time KPI card.
      * Bar chart — complaints by status (Recharts BarChart).
      * Pie chart — complaints by category (Recharts PieChart).

------------------------------------------------------------------------
src/pages/agent/AgentDashboard.jsx
  - AgentDashboard()
      Lists complaints assigned to the authenticated agent
      (GET /api/complaints/assigned).
      - updateStatus(id, newStatus)   PUT /api/complaints/{id}/status; refreshes
                                      the list.
      - uploadPhoto(id)               POST /api/complaints/{id}/attachments as
                                      multipart/form-data.
      Action buttons:
        ASSIGNED    → "Start Work" button sets status to IN_PROGRESS.
        IN_PROGRESS → file picker + "Upload Photo" + "Mark as Resolved" button.
        RESOLVED / CLOSED → "Completed" label.

------------------------------------------------------------------------
src/components/shared/Navbar.jsx
  - Navbar({ links })
      Top navigation bar rendered on every authenticated page.
      Shows the platform brand, an array of navigation links, the current
      user's email, and a Logout button.
      - handleLogout()   Calls logout() from useAuth and navigates to /login.

src/components/shared/StatusBadge.jsx
  - StatusBadge({ status })
      Renders a coloured pill badge for a complaint status value.
      Supports: PENDING, VALIDATED, ASSIGNED, IN_PROGRESS, RESOLVED,
                CLOSED, OVERDUE, REJECTED, CANCELLED.

src/components/shared/StarRating.jsx
  - StarRating({ onSubmit })
      Interactive 1–5 star rating widget with hover highlighting and an
      optional comment textarea. Calls onSubmit({ rating, comment }) when the
      citizen clicks "Submit Feedback" (disabled until a star is selected).

src/components/map/ComplaintsMap.jsx
  - ComplaintsMap({ complaints, center })
      Leaflet map (react-leaflet) centred on Tunisia (default 36.8065, 10.1815).
      Renders a Marker + Popup for each complaint that has latitude/longitude.
      Fixes Vite's broken default Leaflet marker icons via L.Icon.Default.mergeOptions.

------------------------------------------------------------------------
src/index.css
  Global stylesheet. Imports Tailwind CSS v4 (@import "tailwindcss") and
  Leaflet's default CSS (@import "leaflet/dist/leaflet.css").

------------------------------------------------------------------------
Configuration files (frontend root)
  postcss.config.js   Uses @tailwindcss/postcss (Tailwind CSS v4 PostCSS
                      plugin) and autoprefixer.
  tailwind.config.js  Tailwind content paths and theme configuration.

================================================================================
  BACKEND — NEW FILES
================================================================================

------------------------------------------------------------------------
controller/DashboardController.java
  REST controller mapped to /api/dashboard.
  Methods:
  - getStats()
      GET /api/dashboard/stats
      Returns a DashboardStats object with complaint counts by status,
      counts by category, and average resolution time in hours.
  - getNearby(lat, lng, radiusKm)
      GET /api/dashboard/nearby?lat=&lng=&radiusKm=
      Returns complaints within radiusKm kilometres of the given coordinates
      using a PostGIS ST_DWithin geospatial query.

controller/FeedbackController.java
  REST controller mapped to /api/complaints.
  Methods:
  - submitFeedback(id, request, userDetails)
      POST /api/complaints/{id}/feedback
      Allows the citizen who filed a RESOLVED complaint to submit a rating
      (1–5) and optional comment. Transitions the complaint status to CLOSED.

controller/UserController.java
  REST controller mapped to /api/users.
  Methods:
  - getAgents()
      GET /api/users/agents
      Returns all MunicipalAgent records; used by the admin dashboard to
      populate the agent assignment dropdown.

------------------------------------------------------------------------
service/DashboardService.java
  Methods:
  - getStats()
      Aggregates complaint statistics via three repository queries
      (countByStatus, countByCategory, averageResolutionTimeHours) and
      returns a DashboardStats DTO.
  - getComplaintsWithinRadius(lat, lng, radiusKm)
      Converts the radius from kilometres to metres and delegates to
      ComplaintRepository.findWithinRadius.

service/FeedbackService.java
  Methods:
  - submitFeedback(complaintId, request, citizenEmail)   @Transactional
      Validates rating range (1–5), ownership (citizen must match the one
      who filed the complaint), complaint status (must be RESOLVED), and
      duplicate feedback prevention. Persists a Feedback entity, transitions
      the complaint to CLOSED, and returns a FeedbackResponse DTO.

service/FileStorageService.java
  Methods:
  - store(file)
      Saves an uploaded MultipartFile to the configured upload directory
      (application property: file.upload-dir) under a UUID-based filename.
      Returns a relative URL (/uploads/<filename>) suitable for serving
      via WebConfig.

service/UserDetailsServiceImpl.java  (implements UserDetailsService)
  Methods:
  - loadUserByUsername(email)
      Loads a User from the database by email and wraps it in Spring
      Security's UserDetails with a single granted authority equal to the
      user's role name (e.g. ROLE_CITIZEN).

------------------------------------------------------------------------
entity/Feedback.java
  JPA entity mapped to the feedbacks table.
  Fields: id (Long PK), complaint (OneToOne), citizen (ManyToOne),
          ratedAgent (ManyToOne, nullable), rating (Integer),
          comment (TEXT, nullable), submittedAt (OffsetDateTime, set by
          @PrePersist).

entity/Notification.java
  JPA entity mapped to the notifications table.
  Fields: id (Long PK), user (ManyToOne), complaint (ManyToOne, nullable),
          eventType (VARCHAR 60), message (TEXT), isRead (Boolean, default
          false, annotated @Builder.Default), createdAt (@PrePersist),
          readAt (nullable).

------------------------------------------------------------------------
repository/FeedbackRepository.java
  Spring Data JPA repository for Feedback.
  Methods:
  - existsByComplaintComplaintId(complaintId)
      Checks whether feedback already exists for a given complaint UUID
      (used to prevent duplicate submissions).

repository/NotificationRepository.java
  Spring Data JPA repository for Notification.
  Used by SlaMonitoringScheduler to persist SLA_OVERDUE notifications.

------------------------------------------------------------------------
dto/DashboardStats.java
  Lombok @Data @Builder record holding:
  - countByStatus (List<Map<String,Object>>)
  - countByCategory (List<Map<String,Object>>)
  - averageResolutionTimeHours (Double)

dto/FeedbackRequest.java
  Request body for POST /api/complaints/{id}/feedback.
  Fields: rating (Integer, 1–5), comment (String, optional).

dto/FeedbackResponse.java
  Response DTO for feedback submission.
  Fields: id, complaintId, citizenEmail, rating, comment, submittedAt.

------------------------------------------------------------------------
scheduler/SlaMonitoringScheduler.java
  Spring @Scheduled component.
  Methods:
  - checkOverdueComplaints()
      Cron: 0 0 2 * * ? (runs daily at 02:00 AM).  @Transactional.
      Queries all complaints that are ASSIGNED or IN_PROGRESS and past
      their target date. For each:
        1. Sets status to OVERDUE and saves.
        2. Creates a Notification of type SLA_OVERDUE for every admin user.
      Logs summary counts and warnings.

------------------------------------------------------------------------
util/JwtUtil.java
  JWT helper component (HS256 via JJWT).
  Methods:
  - generateToken(email)     Builds and signs a JWT with the email as subject.
  - extractEmail(token)      Parses and returns the subject claim.
  - isTokenValid(token, email)  Checks that the email matches and the token
                                is not expired.
  - isTokenExpired(token)    (private) Compares the expiration claim to now.

------------------------------------------------------------------------
config/JwtAuthFilter.java  (extends OncePerRequestFilter)
  Methods:
  - doFilterInternal(request, response, chain)
      Extracts the Bearer token from the Authorization header, validates it
      via JwtUtil, loads the user via UserDetailsServiceImpl, and sets the
      authentication in the SecurityContext.

config/SecurityConfig.java
  Defines the Spring Security filter chain.
  Permitted (no auth): /api/auth/**, /uploads/**, /v3/api-docs/**,
                       /swagger-ui/**, /swagger-ui.html.
  Role-restricted:
    /api/complaints/*/assign  → ROLE_ADMIN only
    /api/complaints/*/status  → ROLE_AGENT only
    /api/complaints/**        → any authenticated user
  Session policy: STATELESS.

config/WebConfig.java  (implements WebMvcConfigurer)
  Methods:
  - addResourceHandlers(registry)
      Maps /uploads/** to the absolute path of the configured upload
      directory so stored attachment files can be served as static resources.

config/OpenApiConfig.java
  Configures Springdoc OpenAPI (Swagger UI) with a BearerAuth security
  scheme so the Swagger UI can send JWT tokens in test requests.

================================================================================
  POST-SPRINT MODIFICATIONS (April 20, 2026)
================================================================================

1. Exception Handling — GlobalExceptionHandler.java (CREATED)
   - Added @RestControllerAdvice to map exceptions to structured JSON responses:
       { "status": N, "error": "...", "message": "..." }
   - Handles: ResponseStatusException → proper HTTP status,
              DataIntegrityViolationException → 409 Conflict,
              Generic Exception → 500 Internal Server Error

2. AuthService.java (MODIFIED)
   - Registration duplicate email: RuntimeException → ResponseStatusException(CONFLICT)
     returning HTTP 409 "Email already in use"
   - Login failure: wrapped authenticate() in try/catch → ResponseStatusException(UNAUTHORIZED)
     returning HTTP 401 "Invalid email or password"

3. Register.jsx (MODIFIED)
   - Improved error message extraction in catch block to read
     err.response?.data?.message from the structured JSON error response
     instead of showing the generic "Registration failed" fallback.

4. CorsConfig.java (CREATED)
   - Added dedicated CORS configuration bean at
     src/main/java/com/pfa/backend/config/CorsConfig.java
   - Allows origin: http://localhost:5173 (Vite dev server)
   - Allows all methods and headers, with credentials enabled
   - Registers configuration for all routes ("/**")

5. DataSeeder.java (CREATED)
   - Added startup seed data component at
     src/main/java/com/pfa/backend/config/DataSeeder.java
   - Automatically creates baseline categories, one admin, one agent,
     one citizen, and one starter complaint when missing.
   - Seeder is idempotent and enabled by default via property
     app.seed.enabled=true (can be disabled explicitly).

6. scripts/rehearse_demo.sh (CREATED)
   - Added end-to-end rehearsal script covering full lifecycle:
       Citizen login -> create complaint
       Admin login -> assign complaint
       Agent login -> IN_PROGRESS -> RESOLVED
       Citizen feedback -> CLOSED
   - Script supports repeated runs via REHEARSALS variable and fails fast
     if any step returns non-2xx status.
