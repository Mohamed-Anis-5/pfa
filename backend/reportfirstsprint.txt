1) New/Updated Structure Notes
- New folder added: src/test/resources/
- New file in that folder: src/test/resources/application.properties
- File naming fix: src/main/java/com/pfa/backend/service/AuthService.java (renamed from AuthService.Java for Linux compatibility)

2) Folder Relations and Roles
- src/main/java/com/pfa/backend/
  Role: Application source code.
  Relation: This is the runtime core consumed by Spring Boot when the app starts.

  - config/
    Role: Security and request filter configuration (JWT filter chain, auth manager, password encoder).
    Relation: Applied globally to incoming HTTP requests before controller logic.

  - controller/
    Role: HTTP API entry points (e.g., auth endpoints).
    Relation: Receives requests, delegates business logic to service layer, returns responses.

  - dto/
    Role: Request/response payload classes.
    Relation: Used by controller <-> service boundaries for clean data transfer.

  - entity/
    Role: JPA domain models mapped to database tables.
    Relation: Persisted via repository layer; used by service logic.

  - enums/
    Role: Domain constants (status, priority, roles, etc.).
    Relation: Shared by entity/service logic for type-safe values.

  - repository/
    Role: Data access interfaces (Spring Data JPA).
    Relation: Service layer depends on repository to read/write entities.

  - service/
    Role: Business logic (authentication, user details, token generation flow).
    Relation: Called by controllers; coordinates repositories, utils, and security components.

  - util/
    Role: Utility helpers (JWT creation/validation).
    Relation: Used by security filter and auth service.

- src/main/resources/
  Role: Runtime application configuration.
  Relation: application.properties here is used when running normally (currently PostgreSQL-oriented).

- src/test/java/
  Role: Test classes.
  Relation: Validates application context and behavior during mvn test.

- src/test/resources/  [NEW]
  Role: Test-only configuration override.
  Relation: application.properties here shadows main config during test execution so tests can run against H2 without requiring local PostgreSQL.

- target/
  Role: Build output generated by Maven.
  Relation: Compiled classes, processed resources, reports. Not source of truth.

3) Why the New Test Folder Matters
- Without src/test/resources/application.properties, tests inherited main PostgreSQL settings and failed when DB was unavailable.
- With test overrides, mvn test uses H2 and is self-contained.

4) Current Data/Execution Flow (Auth)
- /api/auth/register -> AuthController -> AuthService.register -> UserRepository.save -> JwtUtil.generateToken -> AuthResponse
- /api/auth/login -> AuthController -> AuthService.login -> AuthenticationManager + UserRepository -> JwtUtil.generateToken -> AuthResponse

5) Practical Outcome
- Linux compile issue resolved by correcting file extension case.
- Automated tests no longer depend on external DB availability.
