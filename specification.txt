Specification Coverage Notes
Date: 2026-04-22

Legend
- DONE: implemented in the current repository
- PARTIAL: implemented in part, but does not fully match the specification
- GAP: not implemented or no clear repository evidence found

Overall
- The core complaint lifecycle is mostly implemented end to end.
- The main missing areas are offline submission, map-based fallback when GPS fails, full privacy/compliance coverage, HTTPS/TLS, multilingual support, and operational/performance requirements.
- One important mismatch exists in the feedback workflow: after citizen feedback, the complaint becomes CLOSED in code, while the specification says it should become ARCHIVED.

4.2 Functional Requirements

UC-01 - Soumettre une reclamation
Status: PARTIAL
Done
- Citizen authentication is required for complaint creation.
- Title, category, and description are collected and validated.
- Optional image upload is supported.
- GPS capture is supported.
- Street name fallback is supported when GPS is unavailable.
- Complaint is created with status PENDING.
- A UUID complaintId is generated and returned as the tracking identifier.
Evidence
- backend/src/main/java/com/pfa/backend/config/SecurityConfig.java
- backend/src/main/java/com/pfa/backend/service/ComplaintService.java
- frontend/src/pages/citizen/SubmitComplaint.jsx
Gap
- Offline mode with local save is not implemented.
- No "En attente de synchronisation" workflow or status was found.
- Automatic background synchronization after Internet recovery was not found.
- Manual map selection when GPS fails was not found in the complaint submission flow.

UC-02 - Assigner / Dispatcher une reclamation
Status: DONE
Done
- Admin can assign a complaint to an agent.
- Complaint status changes to ASSIGNED.
- Assignment history is recorded.
- A notification record is created for the assigned agent.
Evidence
- backend/src/main/java/com/pfa/backend/service/ComplaintService.java
- backend/src/main/java/com/pfa/backend/config/SecurityConfig.java
- frontend/src/pages/admin/AdminDashboard.jsx

UC-03 - Traiter et mettre a jour une reclamation
Status: DONE
Done
- Agent can move a complaint from ASSIGNED to IN_PROGRESS.
- Agent can move a complaint from IN_PROGRESS to RESOLVED.
- Agent can add a resolution comment.
- Agent can upload an image as proof.
- resolvedAt and updatedAt timestamps are recorded.
- A notification record is created for the citizen when the complaint is resolved.
Evidence
- backend/src/main/java/com/pfa/backend/service/ComplaintService.java
- backend/src/main/java/com/pfa/backend/controller/NotificationController.java
- frontend/src/pages/agent/AgentDashboard.jsx

UC-04 - Suivre et evaluer le service
Status: PARTIAL
Done
- Citizen can see complaint history.
- Citizen can open a resolved complaint and submit a rating from 1 to 5.
- Optional feedback comment is supported.
- Feedback is stored and duplicate feedback is blocked.
Evidence
- backend/src/main/java/com/pfa/backend/service/FeedbackService.java
- backend/src/main/java/com/pfa/backend/controller/FeedbackController.java
- frontend/src/pages/citizen/MyComplaints.jsx
Gap
- The specification says the complaint should become ARCHIVED after feedback.
- The current code changes RESOLVED to CLOSED after feedback.
- ARCHIVED is currently used by the SLA scheduler for overdue complaints, not by the feedback flow.
